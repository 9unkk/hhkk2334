NMAP 简介

Nmap（“ Network Mapper ”）是用于网络探索和安全审核的开源工具。它设计用于快速扫描大型网络，尽管它可以在单个主机上正常运行。Nmap 以新颖的方式使用原始IP数据包来确定网络上可用的主机，这些主机提供的服务（应用程序名称和版本），它们正在运行的操作系统（和OS版本），包过滤器/防火墙的类型。正在使用中，还有许多其他特性。尽管 Nmap 通常用于安全审核，但许多系统和网络管理员发现它对于日常任务（如网络清单，管理服务升级计划以及监视主机或服务正常运行时间）很有用。

NMAP 功能架构图

Nmap 包含四项基本功能：

  1. 主机发现（Host Discovery）
  2. 端口扫描（Port Scanning）
  3. 版本侦测（Version Detection）
  4. 操作系统侦测（Operating System Detection）

而这四项功能之间，又存在大致的依赖关系（通常情况下的顺序关系，但特殊应用另外考虑），首先需要进行主机发现，随后确定端口状况，然后确定端口上运行具体应用程序与版本信息，然后可以进行操作系统的侦测。而在四项基本功能的基础上，Nmap 提供防火墙与 IDS（IntrusionDetection System,入侵检测系统）的规避技巧，可以综合应用到四个基本功能的各个阶段；另外 Nmap 提供强大的 NSE（Nmap Scripting Language）脚本引擎功能，脚本可以对基本功能进行补充和扩展。

NMAP 主机发现

  • -sP：ping 扫描
nmap -sP 192.168.182.130
  • -P0：无 ping 扫描，通常用来绕过IDS，防火墙
nmap -P0 192.168.182.130
  • -Pn：不检测主机存活，直接对其进行端口探测。
nmap -Pn 192.168.182.130
  • -PS：TCP SYN Ping 扫描
nmap -PS -v 192.168.182.130
  • -PA：TCP ACK Ping 扫描
nmap -PA 192.168.182.130
  • -PU：UDP ping 扫描
nmap -PU 192.168.182.130
  • -PY：SCRIPT INIT Ping 扫描
nmap -PY 192.168.182.130
  • -PE：使用 ICMP echo 扫描
nmap -PE 192.168.182.130
  • -PP：使用 ICMP 时间戳 Ping 扫描
nmap -PP 192.168.182.130
  • -PM：使用 ICMP 地址掩码 ping 扫描
nmap -PM 192.168.182.130
  • -n：永远不对目标 IP 地址作反向解析，加快扫描速度
nmap -n 192.168.182.130
  • -R：永远对目标 IP 作反向解析.一般只有当发现机器正在运行时才进行这项操作。
nmap -R 192.168.182.130
  • –system-dns：使用系统自带的 DNS 解析器
nmap --system-dns 192.168.182.130
  • –traceroute:路由跟踪
nmap --traceroute 192.168.182.130
  • -6：IPv6 扫描
nmap -6 fe80::20c:29ff:fe17:5ae2

NMAP 端口扫描

端口状态

  • open（开放的）

    应用程序正在该端口接收 TCP 连接或者 UDP 报文。

  • closed（关闭的）

    关闭的端口对于 nmap 也是可访问的（它接受 nmap 的探测报文并作出响应），但是没有应用程序在监听。

  • filtered（被过滤的）

    由于包过滤阻止探测报文到端口，Nmap 无法确定该端口是否开放。过滤可能来自专业的防火墙设备，路由器规则或者主机的软件防火墙。

  • unfiltered（未被过滤的）

    未被过滤状态意味着端口可访问，但 nmap 无法确定该端口是开放还是关闭。只有用于映射防火墙规则集的 ack 扫描才会把端口分类到这种状态。用其他类型的扫描如窗口扫描、SYN扫描或者FIN扫描未被过滤的端口可以帮助确定端口是否开放。

  • open|filtered（开放或者被过滤的）

    当无法确定端口是开放还是被过滤的，nmap 就把该端口划分成这种状态。开放端口不响应就是一例。没有响应也有可能意味着报文过滤器丢弃了探测报文或者它引发的任何响应。因此 namp 无法确定该端口是开放还是被过滤的。

  • closed|filtered（关闭或者被过滤的）

    该状态用于 nmap 不能确定端口是关闭还是被过滤的。它只可能出现在 IPID Idle 扫描中。

端口扫描技术

  • -sS（TCP SYN扫描）

    也称为半开放扫描。

    特点：扫描速度快，隐蔽性（扫描时从来不完成 tCP 连接），可靠区分端口的状态

nmap -sS 192.168.182.130

  • -sT（TCP connect()扫描）
    当 SYN 扫描不可用时才用这个。

    特点：扫描慢，会被 IDS、syslog 记录（扫描是会进行完整的 tcp 连接）

nmap -sT 192.168.182.130

  • -sU（UDP 扫描）

    特点：结合 -sS 扫描，扫描慢（使用 –host-timeout 跳过慢速主机）

nmap -sU 192.168.182.130

  • -sN（Null 扫描）

    不设置任何标志位（tcp 标志头是 0）

    特点：比 SYN 扫描更隐蔽，会被 IDS 发现，对 UNIX 系统有用，不能辨别端口状态

nmap -sN 192.168.182.130

  • -sF（FIN 扫描）

    只设置 TCP FIN 标志位

    特点：比 SYN 扫描更隐蔽，会被 IDS 发现，对 UNIX 系统有用，不能辨别端口状态

nmap -sF 192.168.182.130

  • -sX（Xmas 扫描）

    设置 FIN，PSH 和 URG 标志位

    特点：比 SYN 扫描更隐蔽，会被 IDS 发现，对 UNIX 系统有用，不能辨别端口状态

nmap -sX 192.168.182.130

  • -sA（TCP ACK扫描）

    用于发现防火墙规则，确定防火墙是有状态还是无状态的，哪些端口是被过滤的。

nmap -sA 192.168.182.130

  • -sW（TCP 窗口扫描）

    特点：该扫描依赖于互联网上少数系统的实现细节，因此可信度低

nmap -sW 192.168.182.130

  • -sM（TCP Maimon扫描）

    特点：与 Null，FIN，Xmas扫描完全一样

nmap -sM 192.168.182.130

  • –scanflags (定制的TCP扫描)

    可以是 URG， ACK，PSH， RST，SYN，and FIN 的任何组合

nmap --scanflags URGACKPSHRSTSYNFIN 192.168.182.130
  • -sI（空闲扫描）
    特点：扫描方法极端隐蔽，使用僵尸主机对目标进行扫描
nmap -sI 192.168.182.130:80 127.0.0.1
  • -sO（IP协议扫描）
    用来确定目标主机支持哪些 IP 协议（TCP，ICMP，IGMP等等）
nmap -sO 192.168.182.130

  • -b（ftp弹跳扫描）

    它允许用户连接到一台 FTP 服务器，然后要求文件送到一台第三方服务器。这种扫描方式很少被支持，但也是绕过防火墙的好方法

nmap -b 192.168.182.130

NMAP 指纹识别与探测

  • -sV（服务版本探测）

    可同时使用 -A 选项进行操作系统探测和版本探测。

nmap -sV -A 192.168.182.130

  • –allports（扫描所有端口）

    默认情况下，Nmap 会跳过 9100 tcp 端口，指定–allports扫描所有端口

  • –version-intensity < intensity > (设置 版本扫描强度)

    Nmap 扫描强度值在 0~9 之间，默认是 7。值越高，扫描的服务越精确，扫描时间也会更长

  • –version-light (打开轻量级模式)

    这是 –version-intensity 2的方便的别名。轻量级模式使 版本扫描快许多，但它识别服务的可能性也略微小一点。

  • –version-all (尝试每个探测)

    –version-intensity 9的别名， 保证对每个端口尝试每个探测报文。

  • –version-trace (跟踪版本扫描活动)

    这导致Nmap打印出详细的关于正在进行的扫描的调试信息。

  • -O（系统版本探测）

    也可以使用-A来同时启用操作系统检测和版本检测。

  • –osscan-limit (对指定的目标进行操作系统检测)

    采用这个选项，Nmap只对满足这个条件的主机进行操作系统检测，这样可以 节约时间

  • –osscan-guess; –fuzzy (推测操作系统检测结果)

    当Nmap无法确定所检测的操作系统时，会尽可能地提供最相近的匹配，Nmap默认 进行这种匹配，使用上述任一个选项使得Nmap的推测更加有效。

防火墙/IDS躲避和哄骗

  • -f（报文分段）；–mtu（指定偏移大小）

    nmap 在 IP 头后将包分成 8 个字节。因此，一个 20 字节的 TCP 头会被分成 3 个 包，其中 2 个包分别有 TCP 头的 8 个字节，另 1 个包有 TCP 头的剩下 4 个字节。使用两次 -f 可使用 16字节的分段。

    使用 –mtu 选项可 以自定义偏移的大小，使用时不需要 -f，偏移量必须 是 8 的倍数。

namp -f -v 192.168.182.130
namp --mtu -v 192.168.182.130

  • -D（使用诱饵隐蔽扫描）

    指定多个 ip 地址，或者使用 RDN 随机生成几个地址，作为诱饵对目标进行扫描，在指定的诱饵之间使用逗号进行扫描。在进行版本检测或者 TCP 扫描时，诱饵是无效的。

      nmap -D RND:11 192.168.182.130

    随机方法容易被发现，指定几个 IP 对目标主机进行扫描会有更好的效果。

      nmap -D 127.0.0.1,127.0.0.2,127.0.0.253 192.168.182.130

    使用 ME 选项指定自己的真是 IP

      nmap -D 127.0.0.1,127.0.0.2,ME 192.168.182.130

NSE脚本引擎

NMAP 脚本引擎是 NMAP 最强大且灵活的特性。它允许用户编写简单的脚本来自动化执行各种网络任务，基本上这些脚本采用 lua 语言进行编写。通常 NMAP 的脚本引擎可以做很多事情，比如：

漏洞检测

当一个新的漏洞被发现，你想赶在入侵者之前快速扫描网络来识别出有漏洞的系统。虽然 NMAP 不是一个全面的漏洞扫描器，但 NSE 是强大到足以应付苛刻的漏洞检查。许多漏洞脚本已经可用，并计划编写更多的脚本。

后门检测

很多攻击者和一些自动化蠕虫会留下后门以便后期可再次进入。其中一些可以被 NMAP 基于正则表达式检测。

漏洞利用

作为一种脚本语言，NSE 甚至可以进行漏洞利用，而不仅仅是找到它们。添加自定义攻击脚本的这个功能可能对某些人（特别是渗透测试人员）有价值，但不打算把 NMAP 开发成 metasploit 框架那样的东西。
